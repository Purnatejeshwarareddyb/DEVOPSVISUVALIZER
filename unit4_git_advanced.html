üéØ Understanding the Git Lifecycle
Working Directory: Your local files where you make changes

Staging Area (Index): A preparation area for your commits. Use git add to stage changes

Local Repository: Your local .git directory containing commit history. Use git commit to save changes

Remote Repository: Server-hosted repository (GitHub, GitLab). Use git push to upload changes

üí° Pro Tips
Rebase vs Merge: Rebase creates a linear history, merge preserves branch structure

Stashing: Perfect for switching branches without committing incomplete work

Tags: Use semantic versioning (v1.0.0) for releases

Interactive Rebase: Use git rebase -i to edit, squash, or reorder commits
